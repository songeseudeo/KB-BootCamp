# 📅 2026-04-28 학습 기록

## 1. 📖 배운 내용
- **주제**: 제네릭, 컬렉션
- **단어**: 
    - **제네릭(Generic)**: 클래스나 메소드에서 사용할 타입을 객체 생성 시 지정하여 타입 안전성을 높이는 기법
    - **와일드카드(?)**: 제네릭 타입의 범위를 유연하게 지정하는 기법 (상한/하한 제한 가능)
    - **컬렉션(Collection)**: 객체들을 효율적으로 저장, 삭제, 수정할 수 있도록 제공되는 자료구조의 총칭
    - **Map**: 키(Key)와 값(Value)의 쌍으로 데이터를 저장하며, 키의 중복을 허용하지 않는 구조
- **함수**:
    - `<T>`, `<E>`, `<K, V>`: 관례적으로 사용하는 제네릭 타입 파라미터 (Type, Element, Key, Value)
    - `List.of()` / `Map.of()`: 수정 불가능한(Immutable) 컬렉션을 생성하는 메소드
    - `Collections.synchronizedList()`: 멀티 스레드 환경에서 안전하도록 컬렉션을 동기화 처리
- **중요**: 
    - **타입 안전성**: 제네릭을 사용하면 컴파일 시점에 타입 체크가 이루어져 실행 시 발생할 수 있는 `ClassCastException`을 예방함
    - **PECS 원칙**: Producer-Extends, Consumer-Super. 데이터를 꺼낼 때는 상한 와일드카드(`<? extends T>`), 넣을 때는 하한 와일드카드(`<? super T>`) 사용
    - **컬렉션 선택**: 순서가 중요하면 `List`, 중복 제거가 목적이면 `Set`, 키-값 관리가 필요하면 `Map`을 선택

## 3. 🚨 오류 및 해결 (Troubleshooting)
- **문제**: 제네릭 타입의 배열을 `new T[5]`와 같이 직접 생성하려 할 때 컴파일 에러 발생
- **원인**: Java의 제네릭 정보는 런타임에 제거(Erasure)되므로, `new` 연산자는 런타임에 정확한 타입을 알아야 함
- **해결**: `Object` 배열을 생성한 후 타입 변환을 하거나, `Array.newInstance()` 등을 활용하여 우회 해결
