# 📅 2026-04-21 학습 기록

## 1. 📖 배운 내용
- **주제**: 상속
- **단어**: 
    - **상속(Inheritance)**: 부모 클래스의 필드와 메소드를 자식 클래스가 물려받아 코드 중복을 줄이고 유지보수를 쉽게 하는 기능
    - **메소드 오버라이딩**: 부모 클래스의 메소드를 자식 클래스에서 동일한 이름, 매개변수, 반환 타입으로 재정의하는 것
    - **다형성(Polymorphism)**: 부모 타입 변수에 자식 객체를 대입하여, 대입된 객체에 따라 실행 결과가 달라지는 성질
    - **강제 타입 변환(Casting)**: 부모 타입으로 자동 변환된 객체를 다시 자식 타입으로 되돌리는 행위
- **함수**:
    - `super()`: 자식 생성자의 첫 줄에서 부모 클래스의 생성자를 호출할 때 사용
    - `super.메소드()`: 오버라이딩에 의해 숨겨진 부모 클래스의 원래 메소드를 호출할 때 사용
    - `instanceof`: 객체가 특정 클래스의 인스턴스인지 확인하는 연산자 (Java 16부터 패턴 매칭 지원)
- **중요**: 
    - **단일 상속**: Java는 클래스 간 다중 상속을 지원하지 않으며, 단 하나의 부모 클래스만 가질 수 있음
    - **생성자 체이닝**: 자식 객체 생성 시 자식 → 부모 → Object 순으로 생성자가 연쇄 호출됨
    - **final 키워드**: final 클래스는 상속이 불가능하며, final 메소드는 오버라이딩이 불가능함

## 3. 🚨 오류 및 해결 (Troubleshooting)
- **문제**: 부모 클래스에 기본 생성자가 없을 때 자식 클래스에서 컴파일 에러 발생
- **원인**: 컴파일러가 자식 생성자에 자동으로 `super()`를 추가하려 하지만, 부모에게 호출할 기본 생성자가 없음
- **해결**: 자식 생성자 내에서 부모 생성자의 매개변수에 맞춰 `super(매개변수, ...)`를 명시적으로 작성함

## 4. 🌱 오늘의 회고
- **알게된 점**: 모든 클래스의 최상위 부모는 `Object` 클래스이며, `toString`이나 `equals` 같은 주요 메소드를 오버라이딩하여 객체의 상태를 더 정확히 표현할 수 있음을 알게 됨
- **좋았던 점**: `protected` 접근 제한자를 활용해 다른 패키지에 있더라도 자식 클래스라면 부모의 필드에 접근할 수 있는 객체 지향의 유연함을 이해함
