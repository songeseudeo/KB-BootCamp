# 📅 2026-04-21 학습 기록

## 1. 📖 배운 내용
- [cite_start]**주제**:상속
- **단어**: 
    - [cite_start]**상속(Inheritance)**: 부모 클래스의 필드와 메소드를 자식 클래스가 물려받아 코드 중복을 줄이는 기능
    - [cite_start]**메소드 오버라이딩**: 부모의 메소드를 자식 클래스에서 재정의하여 기능을 수정하는 것
    - [cite_start]**다형성(Polymorphism)**: 동일한 타입이지만 대입된 객체에 따라 실행 결과가 달라지는 성질
    - [cite_start]**강제 타입 변환(Casting)**: 부모 타입으로 변환된 객체를 다시 자식 타입으로 되돌리는 것
- **함수**:
    - [cite_start]`super()`: 자식 생성자 첫 줄에서 부모 생성자를 명시적으로 호출
    - [cite_start]`super.메소드()`: 오버라이딩된 상태에서 부모의 원래 메소드를 호출할 때 사용
    - [cite_start]`instanceof`: 객체가 특정 클래스의 인스턴스인지 확인하여 안전한 형변환을 도움
- **중요**: 
    - [cite_start]**단일 상속**: Java는 부모 클래스를 하나만 지정할 수 있는 단일 상속만 지원함
    - [cite_start]**생성자 체이닝**: 자식 → 부모 → Object 순으로 생성자가 연쇄 실행됨
    - [cite_start]**final 키워드**: final 클래스는 상속 불가, final 메소드는 오버라이딩 불가

## 3. 🚨 오류 및 해결 (Troubleshooting)
- [cite_start]**문제**: 부모 클래스에 기본 생성자가 없을 때 자식 클래스에서 컴파일 에러 발생
- [cite_start]**원인**: 부모에게 매개변수 있는 생성자만 있다면 컴파일러가 `super()`를 자동으로 추가하지 못함
- [cite_start]**해결**: 자식 생성자에서 `super(매개변수, ...)`를 명시적으로 호출하여 부모 필드를 초기화함

## 4. 🌱 오늘의 회고
- [cite_start]**알게된 점**: 모든 클래스는 자동으로 `Object`를 상속받으며, `equals`와 `hashCode`를 함께 오버라이딩해야 컬렉션 사용 시 문제가 없다는 것을 배움
- [cite_start]**좋았던 점**: `protected` 접근 제한자를 통해 상속 관계에서의 데이터 보호와 접근 허용의 차이를 명확히 이해함
- [cite_start]**아쉬운 점**: Java 17의 `sealed` 클래스에서 `permits`를 통해 상속 대상을 제한하는 구조가 생소하여 추가 실습이 필요함
