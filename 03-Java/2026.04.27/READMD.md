# 📅 2026-04-27 학습 기록

## 1. 📖 배운 내용
- **주제**: 멀티 스레드
- **단어**: 
    - **프로세스(Process)**: 운영체제에서 실행 중인 프로그램 (독립된 메모리 소유)
    - **스레드(Thread)**: 프로세스 내에서 실행되는 작업 흐름의 단위 (프로세스의 메모리 공유)
    - **동기화(Synchronization)**: 멀티 스레드 환경에서 공유 객체의 데이터를 보호하기 위해 한 번에 하나의 스레드만 접근하도록 제한하는 것
    - **데몬 스레드(Daemon Thread)**: 주 스레드의 작업을 돕는 보조 스레드로, 주 스레드가 종료되면 자동 종료됨
- **함수**:
    - `Thread.sleep(ms)`: 지정된 시간 동안 스레드를 일시 정지 상태로 만듦
    - `synchronized`: 임계 구역을 지정하여 메소드나 블록에 잠금(lock)을 거는 키워드
    - `wait()` / `notify()`: 스레드 간 협업 시 일시 정지 상태로 만들거나 다시 깨울 때 사용
    - `executorService.submit()`: 스레드풀에 작업을 전달하고 실행 요청
- **중요**: 
    - **스레드 생성 방식**: `Thread` 클래스 상속 또는 `Runnable` 인터페이스 구현 (인터페이스 구현 방식이 유연성 면에서 권장됨)
    - **상태 제어**: 스레드는 실행(RUNNABLE), 일시 정지(WAITING/TIMED_WAITING), 종료(TERMINATED) 등의 상태를 가짐
    - **스레드풀**: 스레드의 폭증을 막기 위해 미리 스레드를 만들어두고 제한된 개수로 작업을 관리하는 방식

## 3. 🚨 오류 및 해결 (Troubleshooting)
- **문제**: 두 개의 스레드가 하나의 객체를 공유하여 데이터를 수정할 때 값이 꼬이는 현상 발생
- **원인**: 여러 스레드가 동시에 데이터에 접근하는 '경쟁 상태(Race Condition)' 발생
- **해결**: 데이터 수정 로직이 포함된 메소드에 `synchronized` 키워드를 추가하여 동기화 처리를 함으로써 데이터의 일관성을 확보함

## 4. 🌱 오늘의 회고
- **알게된 점**: 메인 스레드가 먼저 끝나더라도 작업 스레드가 실행 중이면 프로세스는 종료되지 않지만, 데몬 스레드로 설정하면 메인 종료 시 함께 종료된다는 특징을 배움
- **좋았던 점**: 스레드풀 개념을 통해 서버 자원을 효율적으로 관리하는 기본 원리를 이해할 수 있어 유익했음
- **아쉬운 점**: `wait()`와 `notify()`를 이용한 스레드 간 통신 로직이 복잡하여 조건에 따라 정확히 깨우는 부분이 아직 연습이 더 필요함
