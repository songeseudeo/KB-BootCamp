# 📅 2026-04-30 학습 기록

## 1. 📖 배운 내용
- **주제**: 스트림
- **단어**: 
    - **스트림(Stream)**: 컬렉션이나 배열의 요소를 하나씩 참조하여 람다식으로 처리하는 흐름
    - **내부 반복자**: 개발자가 직접 for문을 돌리지 않고, 스트림 내부에서 요소를 반복시키는 방식
    - **파이프라인(Pipeline)**: 여러 개의 스트림이 연결되어 중간 처리와 최종 처리를 수행하는 구조
    - **지연 연산(Lazy Evaluation)**: 최종 처리가 호출되기 전까지 중간 처리는 실제로 수행되지 않는 성질
- **함수**:
    - `filter()`: 조건에 맞는 요소만 걸러냄
    - `map()`: 요소를 다른 형태나 값으로 변환
    - `sorted()`: 요소를 정렬
    - `forEach()`: 최종적으로 요소를 하나씩 소비하며 특정 동작 수행
    - `collect()`: 처리된 결과물(List, Set, Map 등)을 다시 컬렉션으로 수집
- **중요**: 
    - **원본 불변**: 스트림은 원본 컬렉션을 변경하지 않고 읽기만 함
    - **일회용**: 한 번 최종 처리가 끝난 스트림은 재사용할 수 없음 (필요시 다시 생성)
    - **중간 vs 최종**: 중간 처리 메소드는 다시 스트림을 반환하지만, 최종 처리는 결과값을 반환하거나 처리를 종료함

## 3. 🚨 오류 및 해결 (Troubleshooting)
- **문제**: 중간 처리(`filter`, `map` 등)를 작성했는데 아무런 결과가 출력되지 않음
- **원인**: 스트림은 최종 처리(`forEach`, `sum`, `collect` 등)가 호출되어야만 비로소 연산을 시작함
- **해결**: 코드 마지막에 최종 처리 메소드를 추가하여 파이프라인을 완성함
