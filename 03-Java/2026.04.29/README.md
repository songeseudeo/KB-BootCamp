# 📅 2026-04-29 학습 기록

## 1. 📖 배운 내용
- **주제**: 제네릭 복습, 람다식
- **단어**: 
    - **람다식(Lambda)**: 익명 함수를 생성하기 위한 식으로, (매개변수) -> {처리내용} 형태의 간결한 문법
    - **함수형 인터페이스**: 딱 하나의 추상 메소드만 가지는 인터페이스 (`@FunctionalInterface`)
    - **메소드 참조(Method Reference)**: 람다식이 하나의 메소드만 호출하는 경우 `클래스명::메소드명`으로 축약하는 기법
    - **생성자 참조**: 객체를 생성하여 리턴하는 람다식을 `클래스명::new`로 표현하는 기법
- **함수**:
    - `() -> { ... }`: 매개변수가 없는 람다식
    - `(x, y) -> { ... }`: 매개변수가 있는 람다식
    - `() -> expression`: 중괄호와 `return`을 생략한 리턴값이 있는 람다식
- **중요**: 
    - **타겟 타입**: 람다식은 단독으로 쓰일 수 없고 반드시 함수형 인터페이스 타입의 변수에 대입되어야 함
    - **함수형 프로그래밍**: 함수를 데이터 처리부로 전달하여 동작을 파라미터화하는 패러다임
    - **컴파일러 체크**: `@FunctionalInterface` 어노테이션을 붙이면 인터페이스가 두 개 이상의 추상 메소드를 가지지 못하도록 제한함

## 3. 🌱 오늘의 회고
- **알게된 점**: 익명 구현 객체를 만드는 것보다 람다식을 쓰는 것이 코드가 훨씬 간결하고 가독성이 좋다는 것을 깨달음
- **좋았던 점**: `Member::new` 처럼 매개변수 개수가 달라도 인터페이스 형식에 맞춰 적절한 생성자가 호출되는 자바의 타입 추론 능력이 놀라웠음
- **아쉬운 점**: 람다식 내에서 예외 처리를 하거나 로직이 길어질 때 오히려 가독성이 떨어지는 경우가 있어, 적절한 분리 시점에 대한 고민이 더 필요함
