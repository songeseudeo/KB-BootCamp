## 1. 📖 배운 내용
- **주제**: 컴포넌트 심화
- **중요**: 슬롯(구멍)
- **추가 설명**
  - 붕어빵 틀을 부모에게 준다. 부모는 그 안에 팥을 넣을지 슈크림을 넣을지 내용물만 결정해서 다시 자식에게 줍니다. (디자인은 이미 붕어빵 모양으로 정해져 있음)
  - 자식이 부모에게 지금 반죽이 100g 남았다 라는 데이터를 알려준다. 그럼 부모는 그 데이터를 보고 그럼 그 반죽으로 붕어빵을 만들지 말고, 하트 모양 와플(템플릿)로 구워 라고 데이터에 맞춰 디자인까지 바꿔버리는 것
- **단어**
- <Kbd>슬롯</kbd> : 자식의 데이터(내용)와 부모의 템플릿(디자인)을 결합하여, 상황에 맞는 화면을 동적으로 구성하고 싶을 때 사용
- <Kbd>텔레포트</kbd> : 컴포넌트의 논리적 구조는 유지하면서, HTML 요소를 DOM의 다른 위치(예: body)로 렌더링하는 기능

### Options API
1. v-model (양방향 데이터 바인딩)
   * 컴포넌트 간에 데이터를 양방향으로 동기화할 때 사용합니다. 
   * **기본 개념**: 부모의 데이터를 자식에게 전달하고, 자식에서 발생한 변경 사항을 부모에게 즉시 반영합니다.
2. Dynamic Component (동적 컴포넌트)
   * **개념**: 실행 중에 렌더링할 컴포넌트를 동적으로 변경하는 기능
   * **사용법**: `<component :is="컴포넌트명"></component>`
   * **특징**: `:is`에 바인딩된 값에 따라 렌더링되는 컴포넌트가 변경됨
3. KeepAlive (컴포넌트 상태 유지)
   * **개념**: 동적 컴포넌트가 교체될 때 컴포넌트의 상태를 유지하기 위해 사용
   * **특징**: 컴포넌트가 파괴되지 않고 메모리에 캐싱됨 (데이터, 스크롤 위치 유지 등)
4. Provide / Inject (의존성 주입)
   * **개념**: 계층 구조가 깊은 컴포넌트 간에 데이터를 공유하기 위해 사용
   * **원리**: 
    - `Provide`: 상위 컴포넌트에서 데이터를 제공
    - `Inject`: 하위 컴포넌트에서 데이터를 사용
   * **특징**: Props Drilling 문제를 해결하기 위해 사용
5. Teleport (위치 이동 렌더링)
   * **개념**: 컴포넌트의 DOM 요소를 지정한 위치로 이동시켜 렌더링하는 기능
   * **사용법**: `<Teleport to="body">...</Teleport>`
   * **특징**: 모달, 팝업 등 UI 레이아웃 영향을 받지 않아야 하는 경우 사용
6. Async Component & Suspense (비동기 및 로딩 처리)
   * **Async Component**: 필요한 시점에 컴포넌트를 비동기로 로드 (초기 로딩 속도 개선)
   * **Suspense**: 비동기 컴포넌트의 로딩 상태(Loading)를 처리하기 위한 내장 컴포넌트
    - `#default`: 로딩 완료 후 표시할 콘텐츠
    - `#fallback`: 로딩 중 표시할 콘텐츠
