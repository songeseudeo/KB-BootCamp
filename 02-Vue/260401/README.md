# 📅 2026-04-01 학습 기록

## 1. 📖 배운 내용
- **주제**: router를 이용한 라우팅
- **단어**
   - <Kbd>SPA</kbd>: 하나의 페이지 안에 여러 화면을 제공, 화면 전환 시 페이지 전체가 아닌 필요한 부분만 렌더링하는 방식
   - <Kbd>라우팅</kbd>: URI 경로에 따라 컴포넌트를 연결하여 화면에 보여주는 프로세스
   - <Kbd>동적 라우트</kbd>: URI 일부를 파라미터(:id)처럼 사용 ex) 1000명의 회원 정보 페이지를 만들 때 하나의 컴포넌트만 만들고 URL에 포함된 id값만 바꿔 다른 데이터를 보여주는 기능
   - <Kbd>중첩 라우트</kbd>: 컴포넌트 내부에서 또 다른 하위 라우트를 구성하는 방식 (/videos 경로 안에 :id 하위 경로 포함)
    - children 속성을 사용하여 부모-자식 관계를 정의
    - 부모 템플릿 안에 자식 컴포넌트가 나타날 위치에 <router-view></router-view>를 한 번 더 적어줌
   - <Kbd>명명된 라우트</kbd>: 라우트 설정 시 name을 부여하여 이름으로 이동할 수 있음
- **함수**
  - <Kbd>createRouter()</kbd>: 라우터 객체를 생성하는 함수
  - <Kbd>createWebHistory()</kbd>: 브라우저의 뒤로가기, 앞으로가기 버튼을 눌렀을 때 새로고침 없이 URL만 바뀌면서 화면 전환할 거야!
  - <Kbd>useRouter()</kbd>: 컴포넌트 내에서 라우터 인스턴스에 접근하여 프로그래밍 방식으로 페이지를 이동할 때 사용 (router.push())
  - <Kbd>useRoute()</kbd>: 링크 클릭이 아닌 버튼 같은 이벤트 코드로 페이지를 넘길 때 사용 (push, replace, go 등)
- **주요 컴포넌트**
  - <Kbd><RouterView /></kbd>: 현재 URI 경로에 매핑된 컴포넌트가 실제로 렌더링 되어 나타날 위치를 지정
  - <Kbd><RouterLink to="경로" /></kbd>: 클릭 시 다른 경로로 이동할 수 있는 링크태그를 생성, 브라우저의 새로고침 없이 화면 전환
