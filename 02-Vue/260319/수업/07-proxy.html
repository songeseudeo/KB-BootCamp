Vue3 데이터 반응성 실습

1. 개념 이해
model → Vue 밖의 일반 JavaScript 객체
contacts → Vue가 반응형으로 관리하는 상태

같은 모양의 데이터를 사용하더라도, Vue가 관리하는 상태인지 아닌지에 따라 화면 반응이 달라질 수 있다.

2. 현재 화면 (Vue 반응형 상태 출력)
순번 번호 이름 전화번호
{{ index + 1 }} {{ contact.no }} {{ contact.name }} {{ contact.tel }}
현재 연락처 수 : {{ contacts.length }}
3. 일반 객체(model) 변경
예상 결과: model 값은 바뀌지만, 위 표는 자동으로 바뀌지 않는다.
실행 결과 메시지 : {{ modelMessage }}
4. Vue 반응형 상태 변경
예상 결과: 버튼을 누르면 위 표가 바로 업데이트된다.
실행 결과 메시지 : {{ reactiveMessage }}

Vue가 관리하는 반응형 상태를 바꾸면, 화면이 자동으로 업데이트된다.

5. 비교 확인
model 첫 번째 전화번호 : {{ model.contacts[0].tel }}
contacts 첫 번째 전화번호 : {{ contacts[0].tel }}

일반 객체(model)와 Vue 반응형 상태(contacts)의 값이 서로 다르게 유지될 수 있음을 확인한다.

6. 정리
1. Vue는 data()가 반환한 상태를 반응형으로 관리한다.
2. 반응형 상태를 변경하면 화면이 자동으로 업데이트된다.
3. 일반 JavaScript 객체를 변경해도 화면은 자동으로 바뀌지 않는다.
4. Vue3는 내부적으로 Proxy를 사용하여 반응성을 구현한다.
